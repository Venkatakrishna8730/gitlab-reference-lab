GitLab CI/CD + Minikube Complete Setup Guide

Step-by-step ordered documentation to configure GitLab Runner and deploy to Minikube.

STEP 1 — Install GitLab Runner


curl -L --output gitlab-runner https://gitlab-runner-downloads.s3.amazonaws.com/latest/binaries/gitlab-runner-linux-amd64
sudo mv gitlab-runner /usr/local/bin/gitlab-runner
sudo chmod +x /usr/local/bin/gitlab-runner
sudo gitlab-runner install
sudo gitlab-runner start

STEP 2 — Create Runner in GitLab UI

Inside GitLab Project:

  • Settings → CI/CD → Runners
  • Click New Project Runner
  • Select OS → Linux
  • Add Tag → local-shell
  • Click Create Runner

Copy the generated registration command and run it with sudo:


sudo gitlab-runner register --url https://gitlab.com/ --token <REGISTRATION_TOKEN>

During registration provide:

  • Description → my-laptop-runner
  • Executor → shell (IMPORTANT)

STEP 3 — Verify Runner


sudo gitlab-runner verify

You should see:

gitlab-runner: Service is running

In GitLab UI:

  • Runner → Active (Green)

If any issue occurs:


sudo systemctl restart gitlab-runner
sudo gitlab-runner verify

STEP 5 — Add CI/CD Variables

Navigate to:

Settings → CI/CD → Variables

  • DOCKER_USER
    • – Your DockerHub username
  • DOCKER_PASS
    • – Your DockerHub access token
  • KUBE_CONFIG

Get kubeconfig content


cat ~/.kube/config

Copy the entire YAML output and paste it into the KUBE_CONFIG variable.

STEP 6 — Allow Runner Access to Minikube

Option 1 — Quick Permission

            
            sudo chmod -R o+r /home/user/.minikube
sudo chmod -R o+x /home/user/.minikube

Option 2 — Better Approach

    
    sudo chown -R gitlab-runner:gitlab-runner /home/user/.minikube

Option 3 — Safest (Recommended)

    
    sudo usermod -aG $USER gitlab-runner
sudo reboot

STEP 7 — Ensure Minikube Runs Under Runner

            
            sudo su - gitlab-runner 
minikube start --driver=docker

STEP 4 — Create .gitlab-ci.yml

        
        stages:
  - build_push
  - deploy

variables:
  DOCKER_IMAGE: {your_username}/my-k8s-app:latest

build-image:
  stage: build_push
  script:
    - docker build -t $DOCKER_IMAGE .
    - echo "$DOCKER_PASS" | docker login -u "$DOCKER_USER" --password-stdin
    - docker push $DOCKER_IMAGE

deploy-k8s:
  stage: deploy
  # before_script:
  #   - mkdir -p ~/.kube
  #   - echo "$KUBE_CONFIG" > ~/.kube/config
  script:
    - kubectl apply -f k8s/deployment.yaml
    - kubectl apply -f k8s/service.yaml

Paste your pipeline content inside the file.

git add .gitlab-ci.yml git commit -m "Add CI/CD pipeline" git push

Common Errors Faced & Fixes

🔴 Initial Pipeline Not Running

If nothing works and jobs are not executing:

  • Check your internet connection.
  • Avoid using proxy networks.
  • Prefer using a mobile hotspot network.

Tags Issue in .gitlab-ci.yml

If the job stays in Pending:

  • Ensure your .gitlab-ci.yml does NOT contain unmatched tags.
  • If no tags are used, enable:

GitLab → Settings → CI/CD → Runners Enable "Run untagged jobs"

Docker Username Empty Error

If pipeline shows Docker login failure or username empty:

  • Go to Settings → CI/CD → Variables
  • Check DOCKER_USER and DOCKER_PASS values
  • Uncheck the Protected option

kubectl Not Found

If pipeline throws kubectl: command not found:

  • Ensure kubectl is installed on the system.
  • Ensure gitlab-runner has permission to access it.

which kubectl
sudo usermod -aG docker gitlab-runner
sudo systemctl restart gitlab-runner

Also verify access:


sudo su - gitlab-runner
minikube start --driver=docker
kubectl get nodes
